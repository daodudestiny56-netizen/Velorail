# VeloRail: Intent-Driven Financial Gateway for Telegram

VeloRail is a lightweight, low-bandwidth financial gateway that operates as a Telegram bot. It allows users to send money or check balances using plain language text or voice notes. Built for low-bandwidth mobile devices and budget smartphones, VeloRail removes the complexity of traditional banking applications, custom WebViews, and high-friction onboarding processes.

The project is deployed on the Somnia Testnet and integrates Google Gemini for intent parsing, OpenAI Whisper for voice-to-text transcription, and on-chain EVM wallet interactions for real-time asset settlement.

---

## Key Capabilities

1. **Multilingual Intent Parsing**:
   Through integration with Google Gemini 2.5 Flash, VeloRail parses transaction intents in English, Nigerian Pidgin, and Yoruba. The parser extracts the transaction action, numeric amount, input currency, target recipient, and reference details from conversational statements with high accuracy.

2. **Demo Address Book and Name Resolution**:
   To eliminate the need for typing 42-character Ethereum hex addresses on mobile keyboards, VeloRail contains a built-in demo address book. Common names (such as Chidi, Tunde, Alice, and Bob) are resolved automatically to pre-configured EVM addresses.

3. **Multi-Currency and Naira Conversion**:
   The gateway converts local currencies like Nigerian Naira (NGN) or United States Dollars (USD) into the native Somnia Testnet Token (STT) using a demo exchange rate (1 STT = 1,500 NGN). This allows users to request transfers in their local currency while executing the settlement on-chain.

4. **Dynamic On-Chain Dry-Run**:
   Before confirming any transfer, VeloRail estimates the gas fee on-chain using the Somnia Testnet provider. This dry-run acts as an execution guard, detecting insufficient wallet balances or potential transaction reverts before gas is consumed, and presents the user with real-time network fees.

5. **ISO 20022 Compliance**:
   Upon transaction confirmation, VeloRail generates a standardized ISO 20022 Customer Credit Transfer Initiation payload (pain.001.001.09) in JSON format. This aligns the gateway's output with modern global banking communication standards.

6. **Localized Fee Savings Analysis**:
   Every transaction receipt includes a detailed breakdown comparing the cost of a traditional bank wire or remittance service against VeloRail's near-zero gas fee. Fees are presented in the sender's original input currency (e.g., Naira) to demonstrate clear financial benefits.

## Technical Architecture and Directory Structure

The codebase is written in TypeScript and executes as a Node.js process. The directory layout is organized as follows:

* **src/types.ts**: Declares shared TypeScript interfaces, including the transaction intent schema and session cache models.
* **src/session.ts**: Implements an in-memory session manager to track pending transactions with a 5-minute time-to-live cache.
* **src/intentParser.ts**: Handles the Google Gen AI SDK integration, configuring Gemini 2.5 Flash with strict schemas and language parsing rules.
* **src/transcribe.ts**: Manages the binary voice note downloader, duration limits, and OpenAI Whisper transcription API.
* **src/evm.ts**: Orchestrates ethers.js provider connections, bot wallet initialization, address book name mappings, balance queries, gas estimations, and transaction broadcasting on the Somnia Testnet.
* **src/settler.ts**: Handles transaction execution, generates ISO 20022 payment payloads, and structures the final receipt.
* **src/server.ts**: The main entry point initializing the grammY bot framework, command listeners, voice-to-text pipeline, and interactive inline keyboards.

## Setup and Configuration

Follow these steps to configure and run the VeloRail bot locally:

### 1. Prerequisites
Ensure you have Node.js (v18 or higher) and npm installed.

### 2. Environment Variables
Copy the environment template file:
```bash
cp .env.example .env
```

Edit the `.env` file and configure the following variables:
* **TELEGRAM_BOT_TOKEN**: The API token generated by Telegram's @BotFather.
* **GEMINI_API_KEY**: Your API key from Google AI Studio.
* **OPENAI_API_KEY**: (Optional) OpenAI API Key. If left empty, the bot operates in mock transcription mode, utilizing the fallback text.
* **MOCK_TRANSCRIPTION_TEXT**: The default fallback text used when OpenAI Whisper is not configured (set to "Send five thousand naira to Chidi for the generator fuel" for hackathon demos).
* **MAX_VOICE_SECONDS**: The maximum audio length allowed in seconds (default is 30).
* **EVM_RPC_URL**: The RPC endpoint for the Somnia Testnet (defaults to https://api.infra.testnet.somnia.network).
* **BOT_PRIVATE_KEY**: The private key of the bot's funding wallet on the Somnia Testnet.
* **BLOCK_EXPLORER_URL**: The Somnia block explorer transaction URL prefix (defaults to https://explorer-testnet.somnia.network/tx/).

*Warning: Ensure the private key configured belongs to a dedicated testnet wallet. Do not use a wallet containing real mainnet assets.*

## Command Reference

The bot supports the following commands:
* **/start**: Displays the welcome message and usage examples.
* **/help**: Prints system limits and a description of supported actions.
* **/balance**: Queries and displays the native STT balance of the bot's wallet.
* **/cancel**: Clears any pending transaction in the user's session cache.
* **/status**: Displays the number of active sessions and bot uptime.

## Running the Application

Install the package dependencies:
```bash
npm install
```

To run the TypeScript project in development mode:
```bash
npm run dev
```

To build and compile the code to JavaScript for production:
```bash
npm run build
```

To start the compiled server:
```bash
npm start
```
